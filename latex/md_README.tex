 Lex\+Leo is a modular interpreter project written in C.

It explores the design and implementation of a Lisp-\/inspired language while placing strong emphasis on software architecture, testability, and professional engineering practices.

📘 Full documentation\+: \href{https://caltuli.online/lexleo}{\texttt{ https\+://caltuli.\+online/lexleo}}

\DoxyHorRuler{0}
\hypertarget{md_README_autotoc_md2}{}\doxysection{\texorpdfstring{Overview}{Overview}}\label{md_README_autotoc_md2}
Lex\+Leo implements\+:


\begin{DoxyItemize}
\item lexical analysis and parsing,
\item abstract syntax trees (AST),
\item environment-\/based evaluation,
\item runtime value representation,
\item structured memory management.
\end{DoxyItemize}

The project prioritizes clarity, architectural discipline, and correctness over feature breadth.

\DoxyHorRuler{0}
\hypertarget{md_README_autotoc_md4}{}\doxysection{\texorpdfstring{Engineering Focus}{Engineering Focus}}\label{md_README_autotoc_md4}
Lex\+Leo emphasizes\+:


\begin{DoxyItemize}
\item modular architecture with explicit boundaries,
\item dependency injection (CR vs runtime separation),
\item handle-\/based state management,
\item test-\/driven development (TDD),
\item memory abstraction through an OSAL layer,
\item automated documentation with Doxygen.
\end{DoxyItemize}
